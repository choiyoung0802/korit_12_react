```tsx
import { Container, AppBar, Toolbar, Typography, CssBaseline } from "@mui/material"

export default function App() {
  
  return (
    <Container maxWidth='xl'>
    <CssBaseline/>
      <AppBar position="static">
        <Toolbar>
          <Typography variant="h6">
            Car Shop
          </Typography>
        </Toolbar>
      </AppBar>
    </Container>
  )
}
```
- 저희가 다양한 코드에서 확인했듯이 MUI Container 컴포넌트 내에 AppBar 를 하위 컴포넌트로 위치시켰습니다(props drilling 개념 떄문에 매우 중요합니다). position prop 을 이용하여 AppBar 의 위치를 static 으로 잡았습니다.
  - static : 스크롤할 때 앱 바가 상단에 고정되지 않고 사라지게 됩니다.
  - fixed : 스크롤 해서 넘어가더라도 앱 바가 상단에 남아있습니다.

  - maxWidth : 영어 읽으면 답 나오고, 이미 shoppinglist/todolist 에서 사용했습니다(camelCase 인 것만 신경쓰겠습니다).

  - CssBaseline 컴포넌트를 이용하여 여러 브라우저에 리액트 앱의 형태가 균일하게 유지되도록 설정했스빈다. 일반적으로 최상단에 위치시켜 하위 컴포넌트들이 동일한 영향을 받도록 사용합니다.

  - 여기서 제가 AppBar 의 position 을 속성이라고 하지않고 프롭이라고 이야기한 부분에 좀 주목하셔야 합니다. html 에서의 속성은 attribute 였습니다. 하지만 props drilling 에서의 props 의 full word 는 properties 입니다.
